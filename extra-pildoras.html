💊Píldoras: Glosario de conceptos básicos

Este documento recoge pequeñas "píldoras" informativas sobre conceptos básicos que se mencionan a lo largo del curso, y que pueden no explicarse en detalle en los temas principales. Aquí tendrás definiciones claras, directas y sin rodeos, que te servirán como apoyo cuando algo te suene raro en este mundo de acrónimos y palabrejas técnicas.

2FA (Autenticación en dos factores)

2FA significa Autenticación en Dos Factores. Es un sistema de seguridad que requiere dos tipos diferentes de verificación para acceder a una cuenta o servicio.

Ejemplo típico: una contraseña (algo que sabes) + un código en tu móvil (algo que tienes).

Protege mejor que una contraseña sola, porque incluso si la descubren, no podrán acceder sin el segundo factor.

AES (cifrado)

AES (Advanced Encryption Standard) es uno de los algoritmos de cifrado más utilizados en el mundo. Es un sistema de cifrado simétrico, lo que significa que se usa la misma clave para cifrar y descifrar los datos.

Es rápido, seguro y está presente en muchas herramientas modernas como BitLocker, VeraCrypt, 7-Zip, servicios en la nube, y más.

Ejemplo: un archivo comprimido con 7-Zip puede estar cifrado con AES-256 si se protege con contraseña.

Capa 1 - Física

Se encarga de la transmisión física de bits por medios como cables, conectores o señales eléctricas. Es la base del resto de capas.

Ejemplos: cable Ethernet, conectores RJ45, ondas Wi-Fi, fibra óptica.

Capa 2 - Enlace de datos

Establece la comunicación entre dispositivos conectados directamente y detecta errores en la transmisión. Organiza los bits en tramas.

Ejemplos: direcciones MAC, switches, protocolo Ethernet.

Capa 3 - Red

Gestiona el direccionamiento lógico y la entrega de paquetes entre redes distintas. Aquí es donde actúan los routers y se usa la dirección IP.

Ejemplos: IP, ICMP (como en el ping), routers.

Capa 4 - Transporte

Se encarga de asegurar (o no) que los datos lleguen completos y ordenados a la aplicación correcta. Utiliza puertos lógicos y protocolos como TCP o UDP.

Ejemplos: puertos, TCP, UDP.

Capa 5 - Sesión

Establece, mantiene y cierra las sesiones entre aplicaciones. Permite que haya múltiples comunicaciones activas al mismo tiempo.

Ejemplos: control de sesión en streaming, conexión persistente en webs.

Capa 6 - Presentación

Traduce los datos entre el formato de la aplicación y el formato de red. También se encarga del cifrado y la compresión.

Ejemplos: codificación de caracteres, SSL/TLS, compresión de datos.

Capa 7 - Aplicación

Es la capa más cercana al usuario. Aquí viven los programas que usan la red para funcionar.

Ejemplos: navegadores web, clientes de correo, apps de mensajería, FTP.

CLI

CLI son las siglas de Command Line Interface o Interfaz de Línea de Comandos. Es una forma de interactuar con el sistema operativo escribiendo instrucciones en texto, sin usar entorno gráfico.

En la CLI el usuario teclea comandos como:

ls, cd, mkdir, sudo, etc.

Este tipo de interfaz es más potente y flexible que una GUI, especialmente para tareas de administración, automatización o gestión remota. Sin embargo, requiere conocer bien los comandos y su sintaxis.

Tip
Aunque al principio puede parecer más complicada, la CLI permite hacer muchas tareas de forma más rápida y precisa que con la interfaz gráfica.

 

DPI

DPI significa Deep Packet Inspection (inspección profunda de paquetes). Es una técnica que usan los cortafuegos de nueva generación para analizar no solo los encabezados del tráfico de red, sino también el contenido completo del paquete (hasta la capa de aplicación). Esto permite detectar malware, reglas de IDS, patrones de ataque y más.

Endpoint

En ciberseguridad, un endpoint es cualquier dispositivo que se conecta a una red: ordenadores, móviles, impresoras, etc. Cada uno de ellos puede ser un punto de entrada para amenazas. La "seguridad de endpoint" se refiere a proteger estos dispositivos de forma individual y centralizada.

Guest Additions

Traducido como “Complementos de Invitado”, son un conjunto de herramientas que se instalan dentro de una máquina virtual (en sistemas operativos invitados) para mejorar la integración con el sistema anfitrión. Permiten ajustar la resolución automáticamente, compartir el portapapeles, integrar el ratón sin cortes y usar carpetas compartidas entre el sistema anfitrión y huésped.

GUI

GUI son las siglas de Graphical User Interface o Interfaz Gráfica de Usuario. Es la parte visual de un sistema operativo o aplicación que permite al usuario interactuar con el sistema mediante ventanas, iconos, menús y botones, en lugar de tener que usar solo comandos por terminal.

Por ejemplo:

  • Explorador de archivos en lugar de usar ls o cd

  • Gestor de usuarios en lugar de comandos como adduser o usermod

La GUI facilita el uso del sistema a personas sin conocimientos técnicos y acelera tareas cotidianas. Sin embargo, en entornos profesionales también se valora saber trabajar desde línea de comandos (CLI) por su precisión y potencia.

Note
En este curso trabajaremos con ambos enfoques: GUI para tareas comunes y CLI para administración más avanzada.

Hash

Un hash es una especie de "huella digital" de un archivo o cadena de texto. Es un valor único generado por una función matemática (como SHA-256) que representa un contenido específico. Si el contenido cambia, el hash también cambia.

Se usa para:

  • Verificar la integridad de archivos

  • Almacenar contraseñas de forma segura

  • Firmas digitales

ICMP

ICMP significa Internet Control Message Protocol. No se usa para enviar datos como una web o un correo, sino para mensajes de control y diagnóstico. Se utiliza en herramientas como ping y traceroute para comprobar conectividad, errores y tiempos de respuesta en una red.

ICMP tipo 8

En el protocolo ICMP, el tipo 8 corresponde a un mensaje "Echo Request", que es el que se envía cuando se ejecuta un ping. Se espera recibir una respuesta tipo 0 (Echo Reply). Los IDS suelen detectar estos mensajes como parte de escaneos de red o pruebas de conectividad.

Ejemplo: ping 8.8.8.8 envía paquetes ICMP tipo 8 al servidor de Google DNS.

IDS

IDS significa Intrusion Detection System o Sistema de Detección de Intrusos. Es una herramienta que analiza el tráfico de red o los registros del sistema para detectar comportamientos sospechosos o ataques conocidos. No bloquea por sí mismo, solo genera alertas para que un administrador investigue.

Ejemplos: Snort, Suricata, Zeek, OSSEC.

IP

Una IP (Internet Protocol address) es la dirección única que identifica un dispositivo dentro de una red. Puede ser una dirección privada (como las que usan los routers en casa) o pública (como la de un servidor en Internet).

Ejemplo de IP: 192.168.1.10

IPS

IPS significa Intrusion Prevention System o Sistema de Prevención de Intrusos. Además de detectar amenazas como un IDS, el IPS también actúa automáticamente para bloquear el tráfico malicioso, cortar conexiones o aislar dispositivos afectados.

Suele integrarse en soluciones más completas como NGFW o configurarse con herramientas como Suricata o Snort en modo activo.

ISO (imagen de disco)

Una ISO es una imagen exacta de un disco, normalmente de instalación. Contiene todo lo necesario para arrancar e instalar un sistema operativo o un software. Se usa, por ejemplo, para crear una memoria USB de instalación o para montar un sistema en una máquina virtual. Aunque el archivo se llama “imagen”, no es una imagen visual sino una copia estructural de un CD/DVD.

MFA (Autenticación multifactor)

La autenticación multifactor (MFA) es un sistema que añade una capa extra de seguridad al pedir más de una forma de verificación para acceder a un servicio. No basta con solo la contraseña; también necesitas un segundo factor, como un código temporal o tu huella.

Ejemplo: para entrar al correo, además de la contraseña, te piden un código enviado al móvil o generado por una app como Google Authenticator.

Esto protege tu cuenta incluso si alguien descubre tu contraseña.

Modelo OSI

El modelo OSI (Open Systems Interconnection) es un estándar que divide el proceso de comunicación en redes en 7 capas. Cada capa tiene funciones específicas y se apoya en la anterior para enviar y recibir datos correctamente entre dos dispositivos.

Desde el punto de vista práctico, nos ayuda a entender dónde actúan distintas tecnologías (como un firewall o un IDS).

Modo Live

Es un sistema operativo que se carga directamente desde un medio (como una ISO) sin instalarse en el disco. Se utiliza para probar el sistema, hacer tareas de recuperación o, como en nuestro caso, lanzar el instalador de Linux Mint desde un entorno funcional.

NGFW

NGFW significa Next Generation Firewall o Cortafuegos de Nueva Generación. A diferencia de los firewalls clásicos, un NGFW no solo filtra por IP y puerto, sino que también inspecciona el contenido del tráfico (DPI), detecta ataques, bloquea amenazas y permite controlar aplicaciones y usuarios.

Ejemplos: FortiGate, Cisco Firepower, Palo Alto Networks, Sophos XG.

OVA

Un archivo OVA (Open Virtual Appliance) es un formato estándar para exportar máquinas virtuales completas. Permite mover una VM de un equipo a otro de forma sencilla, conservando toda su configuración, disco y estado.

Ping

ping es una herramienta de red que sirve para comprobar si otro dispositivo está accesible a través de la red. Utiliza el protocolo ICMP para enviar un mensaje tipo "¿estás ahí?" (Echo Request) y espera una respuesta (Echo Reply). Si la recibe, el dispositivo está conectado y responde.

1
ping google.com
xxxxxxxxxx
2
1
Haciendo ping a google.com [142.250.184.206] con 32 bytes de datos:
2
Respuesta desde 142.250.184.206: bytes=32 tiempo=27ms TTL=116

Protocolo

Un protocolo es un conjunto de reglas que definen cómo se comunican dos dispositivos a través de una red. Establece el formato, orden y forma de los mensajes para que ambas partes se entiendan, como si fuera el idioma que hablan.

Ejemplos: HTTP (web), FTP (transferencia de archivos), TCP (transporte), IP (direccionamiento).

Puerto

Un puerto es como una "puerta de entrada o salida" que usa un servicio dentro de un dispositivo conectado a la red. Cada servicio (web, correo, FTP...) escucha o se comunica a través de un puerto específico.

Ejemplos:

  • Puerto 80: páginas web (HTTP)

  • Puerto 443: páginas seguras (HTTPS)

  • Puerto 22: acceso remoto por SSH

RSA

RSA es un algoritmo de cifrado asimétrico que utiliza un par de claves: una pública (para cifrar) y otra privada (para descifrar). Es muy utilizado para transmitir datos de forma segura sin compartir la clave previamente.

Ejemplos: conexión segura a una web (HTTPS), firma digital de correos, cifrado de mensajes en servicios como ProtonMail.

SHA

SHA (Secure Hash Algorithm) no es un algoritmo de cifrado, sino de hash. Su función es generar una "huella digital" única de un contenido. Si el contenido cambia, su hash también. Se usa para verificar integridad, guardar contraseñas o firmar datos.

Ejemplo: SHA-256 es ampliamente utilizado en certificados digitales, archivos descargables o blockchain.

Sistema operativo

El sistema operativo (SO) es el software fundamental que gestiona el hardware del ordenador y permite que se ejecuten otros programas. Es lo que permite interactuar con el equipo y coordina tareas como almacenamiento, red, memoria y seguridad.

Ejemplos: Windows 11, Ubuntu, macOS, Android.

Snapshot (Instantánea)

Una snapshot es una copia del estado completo de una máquina virtual en un momento dado. Permite volver a ese estado exacto si algo falla más adelante. Es muy útil antes de hacer pruebas arriesgadas.

sudo

El comando sudo (de superuser do) permite ejecutar tareas con privilegios de superusuario (root) desde una cuenta de usuario normal. Es una forma segura de otorgar permisos elevados solo cuando realmente se necesitan.

Por ejemplo:

xxxxxxxxxx
1
1
sudo apt update

Ejecuta la actualización de repositorios con permisos de administrador.

Al usar sudo, el sistema pedirá la contraseña del usuario actual (no la del root), y si ese usuario está autorizado (pertenece al grupo sudo), podrá ejecutar la orden con privilegios.

Note
El uso de sudo queda registrado en logs del sistema, por lo que también permite auditar qué comandos se ejecutaron como administrador.

TCP

TCP (Transmission Control Protocol) es un protocolo de la capa de transporte que garantiza la entrega ordenada y sin errores de los datos. Establece una conexión entre emisor y receptor (handshake), controla el flujo y retransmite si algo falla.

Ejemplos de uso: navegación web (HTTPS), correo electrónico (SMTP, IMAP), transferencia de archivos (FTP).

UDP

UDP (User Datagram Protocol) es un protocolo de la capa de transporte que no garantiza la entrega. Es más rápido que TCP porque no realiza comprobaciones ni confirmaciones, pero puede haber pérdidas de datos.

Ejemplos de uso: videollamadas, juegos online, streaming en tiempo real.

Virtualización

La virtualización permite ejecutar un sistema operativo dentro de otro mediante software, simulando un equipo completo. Esto se hace mediante una máquina virtual (VM), que actúa como un ordenador independiente pero que funciona desde nuestro sistema principal. Es ideal para pruebas, formación o entornos de laboratorio sin riesgo.

VPN

Una VPN (Virtual Private Network) es una tecnología que permite crear una conexión segura y cifrada entre tu dispositivo y otro punto en Internet, como si estuvieras dentro de una red privada. Se usa para proteger la información que viaja por redes públicas y para acceder a recursos de forma remota.

Ejemplo: conectar desde casa a la red interna de una empresa como si estuvieras allí físicamente.

Whisker Menu

Es el nombre técnico del menú principal que usa el entorno XFCE (como en Linux Mint XFCE). Es el equivalente al "menú de inicio" de Windows. Desde ahí se accede a todas las aplicaciones, configuraciones y opciones del sistema.
