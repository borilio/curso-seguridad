¿Quién protege al protector? Aquí hablamos del sistema operativo, que es como el cerebro (o el mayordomo) que hace que todo funcione bien y seguro.

1. Seguridad en sistemas operativos

Los sistemas operativos son la base de cualquier dispositivo: gestionan recursos, permiten que el software funcione, y son el primer objetivo si alguien quiere comprometer un equipo. Por eso, mantener el sistema seguro no es una opción, ¡es una necesidad!

Important
Si el sistema operativo está comprometido, da igual cuántas herramientas de seguridad pongamos encima. Todo parte de aquí.

Los principales pilares de la seguridad en un sistema operativo son:

  • 🔁Mantenerlo actualizado.

  • ⚙️Configurarlo adecuadamente.

  • 👮Limitar el acceso (usuarios, permisos).

  • 📊Monitorizar servicios, procesos y cambios.

Veamos cómo se gestiona todo esto en los dos sistemas más comunes en el entorno laboral: Windows y Linux.

 

🪟 Windows

  • Es el sistema más utilizado en entornos corporativos y también el más atacado.

  • Es el más usado en sistemas domésticos.

  • Microsoft publica actualizaciones de seguridad regularmente (el famoso Patch Tuesday).

  • Incluye herramientas de seguridad integradas como Windows Defender, BitLocker, y el Firewall de Windows.

  • Soporta políticas de grupo para aplicar configuraciones de seguridad en redes empresariales.

Note
En entornos con Active Directory, la seguridad puede gestionarse de forma centralizada mediante directivas.

 

🐧 Linux

  • Muy usado en servidores y sistemas críticos. Su seguridad depende mucho de cómo se configure.

  • La mayoría de distribuciones publican actualizaciones constantes y permiten automatizar el proceso.

  • El sistema de permisos es muy granular, lo que permite un control preciso de accesos.

  • Existen herramientas avanzadas como AppArmor, SELinux o iptables que permiten endurecer el sistema.

Tip
En Linux, menos es más: mientras menos servicios estén activos, menos superficie de ataque hay.

 

2. Recomendaciones generales de uso seguro

Antes de entrar en configuraciones concretas, es fundamental adoptar una serie de hábitos y buenas prácticas que refuercen la seguridad del sistema operativo. Estas recomendaciones son válidas para cualquier sistema y servirán como base para todo lo que veremos en el curso.

🦸 Superusuario y sudo

En sistemas Linux, el superusuario (root) tiene control total sobre el sistema. Para evitar riesgos, no se trabaja directamente con esa cuenta. En su lugar, los usuarios normales pueden ejecutar tareas administrativas utilizando el comando sudo, que otorga privilegios elevados temporalmente.

Warning
Si algo se puede romper, probablemente suceda ejecutándolo con permisos de superusuario. Por eso solo se debe usar sudo cuando sea estrictamente necesario.

Tip

🤓La palabra sudo viene de “superuser do”, que significa literalmente “superusuario hace”.

 

🧑‍💻 No uses siempre la cuenta de administrador

Tanto en Windows como en Linux, usar una cuenta con privilegios elevados para tareas cotidianas es un riesgo. Si esa cuenta se ve comprometida, el daño puede ser total.

  • En Windows: crea una cuenta estándar y deja la cuenta de administrador solo para tareas puntuales.

  • En Linux: trabaja con tu usuario normal y usa sudo solo cuando realmente lo necesites.

Important
Si un malware se ejecuta con permisos de administrador o root... puede hacer literalmente cualquier cosa en el sistema.

 

🛑 ¡Atención importante para el alumnado! 🛑

Muchas de las configuraciones y acciones que explicamos en este tema sobre Windows no deben aplicarse directamente en los ordenadores del aula.

Por ejemplo, eliminar servicios del inicio o desactivar componentes del sistema puede afectar al funcionamiento del equipo y, si no se sabe bien qué se está haciendo, causar errores graves a estos y al trabajo de quién use estos ordenadores compartidos.

Este contenido tiene como objetivo enseñar lo que se debería hacer en un entorno real para mejorar la seguridad, pero solo debe aplicarse en máquinas virtuales o entornos de prueba.

¡No toques lo que no entiendas aún! Aprenderás a hacerlo, pero paso a paso y en el entorno correcto.

 

3. Configuración básica de seguridad

Un sistema recién instalado no siempre está configurado para ser seguro. La configuración por defecto suele priorizar la funcionalidad sobre la protección. Por eso, uno de los primeros pasos para asegurar un equipo es ajustar su configuración básica.

Las medidas mínimas de seguridad que deberíamos revisar en cualquier sistema operativo incluyen:

  • Desactivar servicios innecesarios.

  • Revisar los programas que se inician con el sistema.

  • Configurar el bloqueo de sesión tras un tiempo de inactividad.

  • Evitar el uso habitual de cuentas con privilegios de administrador.

  • Establecer contraseñas seguras y políticas de expiración.

Warning
Recuerda lo que hemos dicho en el punto anterior, no realizaremos ningún cambio a los equipos reales, solo a las máquinas virtuales.

🪟 Windows

  • Abre el Administrador de tareas con Ctrl + Shift + Esc. En Windows 11, haz clic en "Aplicaciones de arranque" desde el panel izquierdo para gestionar qué programas se ejecutan al iniciar el sistema.

  • Revisa y desinstala aplicaciones innecesarias desde:
    Configuración → Aplicaciones → Aplicaciones instaladas.

  • Para desactivar servicios innecesarios:
    Presiona Win + R, escribe services.msc y desactiva con cuidado solo los que no se usen.

Tip

Hemos abierto cada punto de una forma distinta para ver otras opciones y aumentar la compatibilidad con versiones más antiguas de Windows.

Por ejemplo, los servicios se pueden también comprobar desde la pestaña de “Servicios” en el Administrador de tareas.

Las “aplicaciones de arranque” también se pueden ver en Configuración → Aplicaciones → Inicio.

 

🐧 Linux

Linux también requiere algunos ajustes básicos tras la instalación para reforzar su seguridad, aunque de serie suele venir bastante bien configurado.

  • Bloqueo automático de sesión:
    Ve a Menú → Configuración → Gestor de energía.
    En la pestaña Seguridad, activa la opción "Bloquear automáticamente la sesión cuando se active el salvapantallas".
    Así, cuando el sistema apague la pantalla por inactividad (en la pestaña “Pantalla”, la sesión quedará protegida.

  • Aplicaciones al inicio:
    Accede a Menú → Configuración → Sesión e inicio → Inicio automático de aplicaciones para ver qué programas se ejecutan automáticamente al iniciar el sistema. Desde ahí puedes activar o desactivar los que necesites.

  • Gestión de procesos activos:
    Abre el Gestor de tareas desde el menú. Es el equivalente al Administrador de tareas de Windows: permite ver procesos, uso de CPU y memoria, y finalizar tareas si es necesario.

  • Servicios en segundo plano:
    El sistema ejecuta tareas de fondo (daemons) aunque no se vean. Más adelante veremos cómo se revisan con herramientas como systemctl, pero por ahora basta con conocer su existencia.

Note
Todo esto lo veremos con calma en las prácticas. De momento, lo importante es saber qué partes del sistema se pueden controlar para reforzar su seguridad.

 

 

4. Usuarios y grupos

Una de las mejores formas de proteger un sistema es controlar quién puede hacer qué. Para eso existen los conceptos de usuarios y grupos, que permiten organizar el acceso al sistema y limitar privilegios.

Tip
Un sistema con una buena gestión de usuarios es mucho más seguro que uno donde todo el mundo tiene acceso a todo.

Usuarios → Personas o cuentas con acceso al sistema.
Grupos → Conjuntos de usuarios que comparten permisos comunes.

🪟 Windows

  • Windows permite crear distintos tipos de cuentas: administradores, usuarios estándar, invitados, etc.

  • Puedes gestionar las cuentas desde:
    Configuración → Cuentas → Familia y otros usuarios
    o desde el Panel de control → Cuentas de usuario.

  • También puedes ver o gestionar grupos desde:
    Win + R → lusrmgr.msc (en ediciones Pro).

    Botón derecho en Botón Inicio → Administrador de equipos

Note
En entornos profesionales, se suelen crear grupos como "Ventas", "Contabilidad", "TI" y se asignan permisos según el grupo al que pertenece el usuario.

En Usuarios nos muestra los usuarios del equipo, y si seleccionamos un usuario→ Botón derecho→ Propiedades, podemos:

  • Forzar a que el usuario cambie la contraseña en el próximo reinicio (por si la seguridad del equipo ha sido comprometida y queremos obligar a los usuarios a que cambien la contraseña).

  • Impedir que el usuario cambie la contraseña.

  • Que no expire (no tiene fecha de caducidad).

  • Deshabilitar la cuenta (ya no tendría acceso).

Note
Para cambiar la mayoría de opciones desde aquí, deberemos iniciar el programa con una cuenta con permisos de administrador.

5. Permisos y atributos de archivos

En cualquier sistema operativo es fundamental controlar quién puede acceder, modificar o ejecutar un archivo. Para eso existen los permisos. Estos se aplican tanto a archivos como a carpetas, y determinan lo que cada usuario o grupo puede hacer con ellos.

Tip
Una mala configuración de permisos puede exponer información sensible o permitir que cualquier usuario borre archivos importantes.

 

🪟 Windows

En Windows, los permisos se gestionan desde el entorno gráfico. Cada archivo o carpeta tiene una pestaña de Seguridad donde puedes ver y modificar quién tiene acceso y con qué privilegios.

Pasos para revisar o cambiar permisos:

  1. Haz clic derecho sobre un archivo o carpeta → Propiedades

  2. Ve a la pestaña Seguridad

  3. Ahí verás los usuarios y grupos que tienen permisos sobre ese recurso

  4. Puedes editar permisos como:

    • Leer

    • Escribir

    • Modificar

    • Control total

Note
Solo los usuarios con privilegios elevados (administrador) pueden cambiar permisos de archivos del sistema o de otros usuarios.

De aquí veremos:

  • Nombres de grupos o usuarios: Aquí nos muestra qué grupos o usuarios tienen permisos asignados a esta carpeta.

  • Permisos:

    • Lectura (Read): Te permite ver que el archivo existe y ver su contenido. Si es una carpeta, te permite ver el contenido de la carpeta.

    • Lectura y ejecución (Read & Execute): Incluye Lectura y además permite ejecutar el archivo si es un ejecutable (.exe, .com, etc.). Si se añade este permiso, se añadirá automáticamente Lectura.

    • Mostrar el contenido de la carpeta (List Folder Contents): Es un alias de Lectura y ejecución para una carpeta. Si se selecciona uno, se marcará el otro.

    • Escritura (Write): Permite hacer cambios al contenido del archivo o crear subcarpetas o modificarlas en caso de ser una carpeta.

    • Modificar (Modify): Es un permiso que incluye Lectura y ejecución y Escritura.

    • Control total (Full control): Con este permiso puede hacer lo que quiera sobre el archivo o carpeta sin restricción ninguna.

     

🐧 Linux

Interpretar permisos

En Linux, los permisos están representados por tres letras para cada tipo de usuario:

  • u → usuario (propietario)

  • g → grupo

  • o → otros

Y tres tipos de permiso:

  • r → lectura

  • w → escritura

  • x → ejecución

Puedes ver los permisos con el comando:

xxxxxxxxxx
1
1
ls -l

Ejemplo de salida:

xxxxxxxxxx
1
1
-rwxrw-r-- 1 usuario grupo 1234 abr 18 10:00 documento.txt

Nos centraremos en los primeros caracteres que se ven -rwxrw-r--:

  • -: El primer guion indica que es un archivo normal. Si fuera un directorio, sería una d en lugar de -.

  • rwx: Los primeros 3 caracteres, son los permisos para el propietario del archivo (usuario en el ejemplo). Aquí tiene los 3 permisos read, write y execute.

  • rw-: Los siguientes 3 caracteres, son los permisos del grupo al que pertenece el archivo (grupo en el ejemplo). Aquí tiene sólo read y write. Los usuarios del grupo no pueden ejecutar execute el archivo.

  • r--: Los últimos 3 caracteres, son los permisos para todos los demás usuarios. Solo pueden leer read. No pueden escribir ni ejecutar.

 

Important
¿Tiene sentido dar permiso de ejecución a un archivo de texto?

En general, no. Un archivo de texto normal, como notas.txt, no necesita permiso de ejecución, ya que solo se usa para leer o escribir su contenido.

Sin embargo, si ese archivo contiene código (como un script de Bash, Python, etc.), entonces sí tiene sentido darle permiso de ejecución para poder lanzarlo directamente desde la terminal.

Por ejemplo, si creas un archivo llamado hola.sh con instrucciones de shell, al ejecutar chmod +x hola.sh podrás ejecutarlo con ./hola.sh.

El permiso de ejecución solo debe aplicarse a archivos que realmente lo necesiten. En otros casos, puede considerarse un exceso innecesario e incluso un riesgo si se aplica por error.

 

Cambiar permisos

Para modificar permisos usamos el comando chmod. Se pueden cambiar de varias formas y aunque usaremos una, es importante que veamos las dos ya que es muy común encontrarlas ambas en documentación y aplicadas en sistemas.

🔣Formato simbólico

Primero, debemos escoger para quién vamos a cambiar los permisos:

  • u : El propietario (owner) (el primer trio de bits)

  • g : El grupo al que pertenece el archivo (el segundo trio)

  • o : Otros usuarios (el tercer trio)

Segundo, elegir si queremos añadir o quitar el permiso:

  • + El símbolo suma determina que le agregamos el permiso.

  • - El símbolo resta determina que le denegamos el permiso.

🔢Formato numérico

Además del formato simbólico, también puedes usar notación numérica para cambiar los permisos. A cada permiso se le asigna un valor numérico (como si fueran puntos):

  • r (lectura) = 4

  • w (escritura) = 2

  • x (ejecución) = 1

Para cada grupo (usuario, grupo, otros) sumamos los valores de los permisos deseados:

Los permisos se expresan como un número de tres cifras, donde cada dígito representa los permisos para un tipo de usuario: el primero es para el propietario (u), el segundo para el grupo (g), y el tercero para los demás usuarios (o).

Note

Puede parecer que esto da lugar a confusión, pero no: cada número tiene una única interpretación posible. Es imposible que dos combinaciones distintas den el mismo valor.

Ejemplo:

xxxxxxxxxx
1
1
chmod 755 archivo.sh

Esto asigna los permisos:

  • Usuario: rwx → 7 (4+2+1)

  • Grupo: r-x → 5 (4+1)

  • Otros: r-x → 5 (4+1)

Es decir: el propietario puede leer, escribir y ejecutar; los demás solo leer y ejecutar.

Tip

Esta forma es muy útil cuando quieres aplicar un conjunto de permisos de una sola vez, especialmente en scripts o instrucciones breves.

Note

Note
¿chmod +x o chmod 755?

Ambos comandos pueden usarse para dar permisos de ejecución, pero funcionan de forma diferente:

  • chmod +x saludo.sh → Añade permiso de ejecución sin cambiar los demás permisos.

  • chmod 755 saludo.sh → Establece los permisos exactos: rwxr-xr-x, asegurando que:

    • El propietario puede leer, escribir y ejecutar.

    • El grupo y los demás usuarios pueden leer y ejecutar, pero no modificar.

Usar notación numérica es más preciso y asegura que el archivo tiene solo los permisos necesarios.

 

Cambiar propietario y grupo

Además de los permisos, también se puede cambiar quién es el propietario de un archivo o carpeta, o a qué grupo pertenece. Esto se hace con el comando chown.

Permisos especiales

Aunque normalmente se trabaja con los permisos básicos de lectura, escritura y ejecución, existen también algunos permisos especiales que modifican el comportamiento de archivos o directorios en situaciones concretas. No es necesario dominarlos para usar el sistema con seguridad, pero dejamos aquí una referencia por si quieres profundizar más adelante.

SetUID

A veces, puede pasar que queramos que un usuario pueda acceder a ciertos archivos, sin necesidad de darles permisos superuser. Para ello existe el permiso especial SetUID

Si le indicamos este permiso, un usuario puede ejecutar un archivo, usando el permiso del propietario. Ej: El usuario root es el propietario del archivo /etc/bin/passwd, pero queremos que el usuario maxpower pueda ejecutar este archivo usando el permiso del propietario, que es root.

xxxxxxxxxx
2
1
ls -l /usr/bin/passwd
2
- rws r-x r-x 1 root root 54256 May 16 19:37 /usr/bin/passwd

Vemos que hay una s donde debería haber una x. La s significa SetUID. Y el usuario actual puede ejecutar ese archivo usando los permisos del propietario, que es root.

Veamos como añadir ese permiso especial en nuestro ejemplo de antes:

xxxxxxxxxx
5
1
# Simbólicamente, es s
2
sudo chmod u+s miArchivoMolon.txt
3


4
# Numéricamente, es un 4, que va el primero del resto de permisos 
5
sudo chmod 4755 miArchivoMolon.txt

 

SetGID

Igual que para los usuarios, se puede hacer con los grupos. Es decir, podemos ejecutar un archivo usando los permisos de grupo. Para añadir este permiso especial podemos:

xxxxxxxxxx
3
1
# El archivo /usr/bin/wall tiene activado el GID para el grupo 'tty'
2
ls -l /usr/bin/wall
3
- rwx r-s r-x 1 root tty 27368 Oct 12 13:23 /usr/bin/wall

Vemos como aplicarlo a nuestros archivos:

xxxxxxxxxx
9
1
# Simbolicamente, es g
2
sudo chmod g+s miArchivoMolon.txt
3


4
# Numéricamente, es 2
5
sudo chmod 2755 miArchivoMolon.txt
6


7
# Comprobamos
8
ls -l miArchivoMolon.txt
9
- rwx r-s r-x 1 maxpower grupo_aburrido 0 Oct 12 13:23 miArchivoMolon.txt

Cuando alguien intente ejecutar este archivo, el mismo se ejecutará con los permisos del grupo grupo_aburrido, y no con los permisos que tenga el usuario que lo ejecuta.

 

Sticky Bit

Este permiso especial suele usarse en carpetas donde todo el mundo tiene acceso, y queremos que un usuario no pueda borrar archivos/carpetas de otro usuario, ya que todos tienen permiso de escritura. Suele usarse en las carpetas /tmp y /var/tmp.

xxxxxxxxxx
3
1
# Veamos los permisos de la carpeta /tmp (con d vemos solo el directorio y no su contenido)
2
ls -ld /tmp
3
d rwx rwx rwt 11 root root 4096 Oct 12 13:23 /tmp

Fíjate que tiene todos los permisos, y en el de ejecución para otros hay una t. Es el permiso especial de sticky (pegajoso). Significa que todos pueden agregar información (crear y modificar archivos y carpetas), pero solo root podrá borrar la carpeta /tmp.

Podemos agregar el permiso sticky bit de forma simbólica o numérica:

xxxxxxxxxx
9
1
# Simbólicamente
2
sudo chmod +t miCarpeta/
3


4
# Numéricamente
5
sudo chmod 1755 miCarpeta/
6


7
#Comprobamos...
8
ls -ld miCarpeta/
9
d rwx r-x rwt 11 maxpower grupo_aburrido 4096 Oct 12 13:23 miCarpeta/

6. Seguridad en servicios y procesos

Un sistema operativo no solo ejecuta las aplicaciones que abrimos: también pone en marcha muchos procesos en segundo plano (servicios o "daemons") que realizan tareas automáticamente, como gestionar redes, imprimir, sincronizar la hora o actualizar el sistema.

Aunque muchos de estos servicios son necesarios, algunos pueden ser innecesarios o incluso suponer un riesgo si se mantienen activos sin control. Por eso es importante saber:

  • Qué procesos se están ejecutando

  • Qué servicios arrancan automáticamente al iniciar

  • Cuáles se pueden desactivar de forma segura

Important
Cuantos más servicios innecesarios estén activos, mayor es la superficie de ataque. Desactivar lo que no se usa es una buena práctica de seguridad.

🪟 Windows

En Windows, los servicios del sistema se pueden gestionar desde una herramienta llamada Servicios, donde puedes ver su estado y configurarlos para que se inicien (o no) con el sistema.

Abrir la herramienta de servicios:

  • Pulsa Win + R → escribe services.msc y pulsa Intro

  • También puedes buscar “Servicios” desde el menú de inicio.

  • O la pestaña “Servicios” desde el administrador de tareas (Control + Mayús + ESC).

Una vez dentro, puedes:

  • Ver la lista completa de servicios

  • Ver su estado (en ejecución, detenido, etc.)

  • Cambiar su tipo de inicio:

    • Automático: se inicia al arrancar Windows

    • Manual: solo se inicia si una aplicación lo necesita

    • Deshabilitado: no se ejecuta

Note
Solo deberías desactivar servicios si sabes para qué sirven. Algunos están relacionados con la red, impresoras, actualizaciones o seguridad. Mejor no tocarlos si no lo tienes claro.

🧪 Práctica: detener servicios en Windows

Objetivo: Localizar un servicio del sistema, detenerlo temporalmente, comprobar su efecto, y volver a iniciarlo.

Servicio usado: Cola de impresión
(Este servicio se encarga de gestionar la impresión. Si no hay impresoras instaladas, es seguro detenerlo).

1. Abrir la herramienta de servicios

  • Pulsa Win, escribe “Servicios” y haz clic derecho sobre el resultado → Ejecutar como administrador

  • También puedes pulsar Win + R, escribir services.msc, y luego pulsar Ctrl + Shift + Intro para abrirlo con privilegios elevados

  • Busca en la lista el servicio llamado Cola de impresión

  • Haz doble clic para abrir sus propiedades

2. Revisar su estado actual

  • Verifica que el estado del servicio sea En ejecución

  • El tipo de inicio suele ser Automático

3. Detener el servicio

  • Haz clic en el botón Detener

  • Pulsa Aplicar y luego Aceptar

4. Comprobar el efecto

  • Abre el panel de impresoras (Inicio → Configuración → Dispositivos → Impresoras y escáneres)

  • Aunque no tengas impresoras físicas conectadas, lo más seguro es que tengas algunas virtuales, como Microsoft Print to PDF o similares.

  • Verás que las impresoras desaparecen o muestran un estado de error

Note
Si no tienes ninguna impresora instalada, el panel puede quedarse en blanco. Esto es normal y significa que el servicio fue detenido correctamente.

5. Reiniciar el servicio

  • Vuelve a la ventana del servicio Cola de impresión

  • Haz clic en Iniciar

  • Confirma que las impresoras vuelven a aparecer o se restablecen

Tip
Esta es una forma habitual de solucionar problemas con impresoras que no responden. Reiniciar el servicio "Cola de impresión" puede arreglar muchos errores de forma rápida.

7. Instalación de software en Windows y Linux

Antes de continuar explorando herramientas integradas de protección, es importante entender cómo se instala nuevo software de forma segura en cada sistema operativo. No siempre es necesario buscar archivos en páginas web: tanto Windows como Linux tienen formas oficiales y controladas de instalar programas.

🪟 Windows

En Windows, además de los instaladores clásicos .exe o .msi, existe una tienda oficial llamada Microsoft Store, desde la cual se pueden descargar muchas aplicaciones de forma segura.

¿Cómo abrir la Microsoft Store?

  • Pulsa Win y escribe “Microsoft Store”

  • O haz clic en su icono desde el menú de inicio

Aquí puedes buscar programas como antivirus, navegadores, utilidades, etc. La instalación es sencilla: solo hay que hacer clic en “Obtener” y se instalarán automáticamente.

Note
No todo el software está en la tienda, pero para muchas tareas básicas es una opción segura y rápida que evita buscar instaladores dudosos por internet.

🐧 Linux

En Linux, el software se instala principalmente desde los repositorios oficiales, que son como tiendas centralizadas controladas por la distribución.

Para instalar programas desde la terminal se usa el siguiente comando:

xxxxxxxxxx
1
1
sudo apt install nombre_del_paquete

¿Qué significa esto?

  • apt es el gestor de paquetes en distribuciones basadas en Debian, como Linux Mint o Ubuntu

  • install indica que quieres instalar algo

  • nombre_del_paquete es el nombre exacto del programa (cada programa tiene un nombre único identificativo)

  • sudo se usa para obtener permisos de administrador (como ya deberías de saber )

Ejemplo:

xxxxxxxxxx
1
1
sudo apt install vlc

Warning
No es necesario descargar nada desde internet: el sistema buscará automáticamente el paquete, lo descargará y lo instalará desde sus servidores oficiales.

 

¿Y si quiero desinstalar un programa?

También puedes eliminar un programa que hayas instalado con:

xxxxxxxxxx
1
1
sudo apt remove nombre_del_paquete

Ejemplo:

xxxxxxxxxx
1
1
sudo apt remove vlc

 

Tip
Este comando desinstala el programa, pero deja sus archivos de configuración. Si quieres borrarlo todo por completo, puedes usar sudo apt purge en lugar de remove.

 

Otras formas de instalar software en Linux Mint:

  • Desde el Gestor de software (accesible desde el menú)

 

Tip
A diferencia de Windows, en Linux se prioriza la instalación desde repositorios. Esto evita problemas con archivos inseguros o versiones incompatibles.

🧪 Práctica: bloquear temporalmente el tráfico con GUFW

Objetivo: Evitar el acceso a páginas web bloqueando los protocolos HTTP y HTTPS de forma sencilla y comprensible.

  • Rellena los campos así:

    • Política: Denegar

    • Dirección: Ambos

    • Categoría: Red

    • Subcategoría: Servicios

3. Comprobar el resultado

  • Abre tu navegador e intenta acceder a cualquier página web.

  • Verás que no se carga ninguna, ni siquiera páginas https como google.com.

  • Por otro lado, comprueba que funciona un ping a la página que estás probando:

    xxxxxxxxxx
    1
    1
    ping google.com

  • Para detener el envío de señales ping, pulsa Control + c.

Note
Si alguna web carga, comprueba que las reglas están activas y bien definidas. En algunos casos el navegador puede mostrar una página en caché.

4. Desactiva el firewall temporalmente.

  • En la ventana del firewall, pulsa el botón de Estado para apagarlo. Puede tardar unos segundos en responder. Se paciente.

  • Cuando el firewall está apagado, volverás a tener conexión. Compruébalo.

  • Vuelve a habilitarlo para dejarlo encendido.

5. Eliminar las reglas para recuperar la conexión

  • En la pestaña Reglas, selecciona las reglas de HTTP y HTTPS que creaste.

  • Pulsa el icono de ➖ (menos) para eliminarlas.

  • Una vez que elimines las reglas del firewall, el navegador volverá a tener conexión y mostrará las páginas.

Caution

Fíjate que todos los eventos han quedado registrado en la pestaña de Registro, y pueden ser revisados por un administrador. Todo deja huella.

Tip
Esta práctica demuestra cómo un firewall puede bloquear tipos de tráfico conocidos, sin necesidad de saber el número de puerto que usan.

 

 

8.2 ClamAV – Antivirus bajo demanda

Linux no requiere antivirus activo en la mayoría de entornos, pero ClamAV es útil si recibes archivos de otras personas o usas memorias USB.

Instalación:

Ya deberíamos tener ClamAV instalado de la práctica de antes. Si no, puedes instalarlo así:

Comprobar instalación:

🧪 Práctica: creando un virus en Linux 🦠

Objetivo: Simular un archivo malicioso (sin peligro real) y comprobar si el antivirus ClamAV lo detecta correctamente.

1. Crear el archivo desde un editor de texto

  1. Abre un editor de texto gráfico (por ejemplo, Mousepad, Xed, o el que tengas en Mint).

  2. Copia exactamente esta línea de texto y pégala tal cual:

    xxxxxxxxxx
    1
    1
    X5O!P%@AP[4\PZX54(P^)7CC)7}$EICAR-STANDARD-ANTIVIRUS-TEST-FILE!$H+H*

  3. Guarda el archivo como eicar.txt en tu carpeta Descargas.

🤓 Opción alternativa: crear el archivo desde la terminal

Si prefieres hacerlo desde la consola (la consola siempre mola mucho más), puedes usar este comando:

xxxxxxxxxx
1
1
echo 'X5O!P%@AP[4\PZX54(P^)7CC)7}$EICAR-STANDARD-ANTIVIRUS-TEST-FILE!$H+H*' > ~/Descargas/eicar.txt

Note
El símbolo > sirve para redirigir la salida del comando y guardarla en un archivo. En este caso, creamos un archivo llamado eicar.txt con el contenido del “virus simulado”.

 

2. Analizar el archivo con ClamAV

Abre la terminal y ejecuta:

xxxxxxxxxx
1
1
clamscan ~/Descargas

Verás un resultado como este:

xxxxxxxxxx
1
1
/home/usuario/Descargas/eicar.txt: Eicar-Test-Signature FOUND

Warning

Recuerda que ClamAV no va a hacer nada, tan solo nos avisa de que ha detectado un virus. Nosotros deberíamos borrar tomar las medidas pertinentes.

No lo borres, déjalo un ratito más con nosotros 😷.

 

Extra: Añadir una GUI a ClamAV

¿Qué es ClamTk?

  • Permite hacer análisis de archivos y carpetas desde un explorador gráfico

Desde ahí podrás:

  • Seleccionar carpetas o archivos para analizar

  • Revisar los resultados

  • Volver a analizar el archivo eicar.txt si todavía lo tienes en Descargas

Note
Esta herramienta no añade análisis en tiempo real, pero es útil si prefieres no trabajar con comandos.

Important

En este caso lo hicimos por pasos para comprender mejor cómo ampliar un programa base con nuevas funcionalidades (interfaz gráfica, daemon en segundo plano, etc.).

9. Buenas prácticas de mantenimiento

Un sistema seguro no es solo aquel que tiene antivirus o cortafuegos activados. También es aquel que se mantiene al día, se limpia periódicamente y se revisa con sentido común. Un sistema abandonado o lleno de basura puede ser igual de vulnerable que uno mal configurado.

En este apartado veremos algunas recomendaciones clave para el mantenimiento básico tanto en Windows como en Linux.

 

🪟 Windows

  1. Mantener el sistema actualizado

    • Usa Windows Update regularmente para aplicar parches de seguridad y mejoras.

  2. Revisar las aplicaciones de inicio

    • Evita que se acumulen programas innecesarios en el arranque.

    • Usa el Administrador de tareas > pestaña Aplicaciones de arranque.

    Note
    Ya vimos cómo revisar y desactivar aplicaciones al inicio en el punto 3.

  3. Eliminar programas que no se usan

    • Desde Configuración > Aplicaciones, desinstala lo que no sea necesario.

  4. Liberar espacio

    • Usa la herramienta de Liberador de espacio en disco o Configuración > Almacenamiento.

  5. Revisar errores del sistema

    • Ejecuta periódicamente sfc /scannow en PowerShell para comprobar archivos del sistema.

      xxxxxxxxxx
      1
      1
      sfc /scannow

    • Ejecuta la herramienta de diagnóstico avanzada que analiza el SO en busca de corrupción en sus archivos.

      xxxxxxxxxx
      5
      1
      # Escanea sin reparar
      2
      DISM.exe /Online /Cleanup-image /Scanhealth
      3
      

      4
      # El siguiente paso intenta reparar los daños detectados
      5
      DISM.exe /Online /Cleanup-image /Restorehealth

     

  6. Hacer copias de seguridad

    • Usa herramientas como Historial de archivos o sistemas externos de copia.

Note
Veremos en profundidad el tema de copias de seguridad más adelante en el curso, con ejemplos prácticos y herramientas específicas.

 

🐧 Linux

  1. Actualizar el sistema regularmente

    • Ejecuta sudo apt update && sudo apt upgrade o usa el gestor gráfico de actualizaciones.

      xxxxxxxxxx
      1
      1
      sudo apt update && sudo apt upgrade

    ¿Qué hace exactamente?

    • sudo apt update: actualiza la lista de paquetes disponibles. Es decir, consulta en los repositorios si hay nuevas versiones de los programas o componentes del sistema.

    • sudo apt upgrade: instala las actualizaciones disponibles para los programas que ya tienes en tu sistema, pero sin eliminar ni instalar paquetes nuevos que puedan causar cambios importantes.

    El && une los dos comandos: primero se actualiza la lista, y solo si eso termina correctamente, se lanzan las actualizaciones.

    Tip
    Es recomendable ejecutar este comando al menos una vez por semana, o antes de instalar paquetes nuevos, para evitar conflictos o errores por versiones antiguas.

    Important
    A diferencia de Windows, en Linux el comando sudo apt update no solo actualiza el sistema operativo, sino también todas las aplicaciones de terceros instaladas desde los repositorios.
    Es decir, actualiza el sistema completo (navegadores, utilidades, herramientas de red, etc.) en un solo paso, sin necesidad de buscar actualizaciones aplicación por aplicación.

     

  2. Eliminar paquetes innecesarios

    • Usa sudo apt remove para desinstalar programas que ya no necesitas.

    • Puedes limpiar dependencias huérfanas con sudo apt autoremove.

    Note
    Ya explicamos cómo instalar y desinstalar programas en el punto 7.

     

  3. Verificar el espacio en disco

    En Linux es importante comprobar periódicamente cuánto espacio está usando el sistema o si una carpeta concreta está creciendo demasiado.

    💾 Ver el uso general del disco:

    Comando para mostrar todas las particiones montadas con su uso:

    xxxxxxxxxx
    1
    1
    df -h

    Este comando muestra el tamaño total, el espacio usado, el espacio libre y el punto de montaje. La opción -h significa "human-readable", es decir, en MB/GB.

    📂 Ver cuánto ocupa cada carpeta del directorio actual:

    Comando para listar el tamaño de todas las carpetas (sin entrar recursivamente en cada una):

    xxxxxxxxxx
    1
    1
    du -sh *

    Esto te permite ver qué carpetas ocupan más espacio dentro de la ruta en la que estás. Muy útil para detectar carpetas “gigantes”.

    Tip

    Al usar * al final del comando, le decimos que muestre todas las carpetas. Si queremos una carpeta concreta, le indicamos la carpeta concreta. Por ejemplo:

    xxxxxxxxxx
    1
    1
    du -sh Descargas

     

  4. Limpiar la caché de paquetes

    Cada vez que instalas o actualizas un programa con apt, el sistema descarga un archivo .deb y lo guarda temporalmente en una carpeta llamada caché de paquetes. Con el tiempo, esto puede ocupar bastante espacio, especialmente si haces muchas actualizaciones.

    Para liberar ese espacio, puedes usar:

    xxxxxxxxxx
    1
    1
    sudo apt clean

    Este comando borra todos los archivos .deb almacenados en caché en /var/cache/apt/archives. No afecta a los programas ya instalados: solo elimina los instaladores que ya no se necesitan.

    Tip
    Es completamente seguro y no afecta al funcionamiento del sistema. Se recomienda hacerlo de vez en cuando si el espacio en disco es limitado.

     

  5. Revisar procesos y servicios activos

    • Usa htop o el Gestor de tareas para ver qué se está ejecutando.

    • Usa systemctl list-units --type=service para ver servicios activos.

    Note
    Ya trabajamos con servicios y procesos en el punto 6.

     

    Tip
    Una característica muy útil de algunas distribuciones como Linux Mint o Ubuntu es que, cuando intentas usar un comando que no está instalado, el sistema puede sugerirte automáticamente cómo instalarlo.

    Por ejemplo, si escribes htop y no está instalado, puede aparecer este mensaje:

    xxxxxxxxxx
    2
    1
    No se ha encontrado la orden «htop», pero se puede instalar con:
    2
    sudo apt install htop

    Esto te ayuda a descubrir herramientas nuevas y a instalarlas fácilmente sin necesidad de buscarlas por internet.

     

  6. Realizar copias de seguridad

    • Aunque no viene activado por defecto, puedes usar herramientas como Timeshift o soluciones externas.

      Note
      Veremos en profundidad el tema de copias de seguridad más adelante en el curso, con ejemplos prácticos y herramientas específicas.
